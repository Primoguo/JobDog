**版本**：2.1

**目标平台**：iOS 17.0+（适配 iPhone SE 至 iPhone 15 Pro Max）

## 一、产品概述

### 1.1 产品定位

Job Dog 是一款面向当代年轻人和中年人的**情感陪伴类 App**。在信息过载、快节奏生活的当下，用户打开 Job Dog 不是为了打卡、不是为了完成任务，而是为了**逃离千篇一律的快娱乐环境**，获得真正的放松和简单的陪伴。

**核心概念：拟人化狗狗**
- 狗狗**外形是狗**（各种犬种），但**行为完全拟人化** — 像人类一样上班、做饭、通勤、健身
- 风格参考：动物森友会村民 — 是动物但过着人类生活
- **不是宠物模拟**：不是养一只真狗，而是看一个"狗人"过自己的生活
- 场景中只有主角狗狗一只，其他角色是 NPC（同事、路人等）

### 1.2 核心理念

| 维度 | 传统习惯类 App | Job Dog v2.0 |
|------|---------------|-------------|
| 驱动力 | 任务驱动、目标导向 | 情感驱动、陪伴导向 |
| 用户感受 | "我应该做..." | "我想看看我的狗..." |
| 交互频率 | 定时打卡 | 随时打开，随时陪伴 |
| 情感连接 | 弱（工具属性） | 强（情感投射） |
| 核心体验 | 成就感 | 放松感、被需要感 |

### 1.3 一句话描述

> **Job Dog 是你的数字宠物，也是你的生活镜像。它不需要你做什么，只是陪着你。**

## 二、目标用户

### 2.1 用户画像

**主要用户群体**：25-45 岁的城市青年和中年人

| 用户类型 | 特征 | 核心需求 |
|---------|------|---------|
| 上班族 | 朝九晚六，生活规律但千篇一律 | 下班后的放松，回家有人"等"的感觉 |
| 自由职业/无业 | 生活无拘无束，但可能感到迷茫 | 稳定的陪伴感，日常的小确幸 |
| 自律型 | 严格作息，爱健身/徒步等 | 共同成长的伙伴，不评判的陪伴 |

### 2.2 用户场景

**场景 1：上班族下班回家**
- 疲惫地打开 App
- 狗狗根据时间做出反应（可能是刚睡醒、可能在发呆、可能在门口等你）
- 简单互动，摸摸头，喂点吃的
- 感受：有人等我回来了

**场景 2：自由职业者午后**
- 工作间隙打开 App
- 狗狗可能在晒太阳、可能在玩耍
- 一起"出去走走"（散步功能）
- 感受：有个伴儿真好

**场景 3：自律型用户晨跑后**
- 运动完打开 App
- 狗狗也"刚运动完"（状态同步）
- 一起喝水、休息
- 感受：我们一起在生活

### 3.2 首次体验流程（Onboarding）

#### 完整流程

```
欢迎页 → 生活模式选择 → 性别选择 → 随机生成狗狗 → 领养仪式（命名）→ 主界面
```

**关键原则**：
- 犬种**不可选**，完全随机生成
- 性别选择**只影响装扮风格**，不影响犬种/颜色
- 一旦确认领养，狗狗**不可更改**（命运感）
- 支持存档，下次打开自动加载

---

#### 3.2.1 欢迎页

**界面元素**：
- APP Logo（动森风格插画）
- 粗体 "Job Dog" 标题
- "开始" 按钮
- 如有存档，显示"继续游戏"选项

**交互**：
- 点击"开始" → 进入生活模式选择
- 点击"继续游戏" → 直接进入主界面（跳过 Onboarding）

---

#### 3.2.2 生活模式选择

**设计原则**：不是"你是什么人"，而是"你希望你的生活节奏是怎样的"

**交互设计**：
- 三个卡片**纵向堆叠**排列
- 每个卡片包含：图标 + 名称 + 描述 + 小场景预览（动森风格）
- 点击卡片选中，可返回修改

| 模式 | 图标 | 描述文案 | 狗狗特征倾向 |
|------|------|---------|-------------|
| 🏢 上班族 | 公文包 | 朝九晚六，规律但安稳 | 狗狗更规律，会在固定时间等你 |
| 🌊 自由人 | 冲浪板 | 随性而为，不被定义 | 狗狗更随性，随时可能做奇怪的事 |
| 🏃 自律者 | 跑鞋 | 严格作息，追求更好的自己 | 狗狗更活力，喜欢运动和挑战 |

---

#### 3.2.3 性别选择

**说明**：性别**不是狗狗的性别**，而是决定狗狗的**装扮风格方向**。

| 选择 | 装扮风格 | 默认装扮 |
|------|---------|---------|
| 男孩 | 偏男性化 | 领带/棒球帽 |
| 女孩 | 偏女性化 | 蝴蝶结/贝雷帽 |

**影响范围**：
- ✅ 影响：默认装扮、可解锁的装扮池
- ❌ 不影响：犬种、毛色、性格、行为

---

#### 3.2.4 随机生成狗狗（核心创新点）

**完全随机生成**：
- 犬种：从 12 个犬种中随机
- 毛色：从品种标准色库中随机
- 眼睛颜色：随机（小概率异色瞳）
- 特殊标记：小概率出现斑点/花纹
- 初始装扮：从**性别对应的装扮池**中随机

**12 个犬种池**（选择标准：像素画视觉辨识度高、体型差异明显）：

| 类别 | 犬种 | 体型 | 视觉特征 |
|------|------|------|---------|
| 小型犬 | 柴犬、柯基、比熊、博美 | 小 | 柴犬三角耳/柯基短腿大屁股/比熊棉花糖/博美毛球 |
| 中型犬 | 边牧、哈士奇、萨摩耶、法斗 | 中 | 边牧双色毛/哈士奇蓝眼面具/萨摩耶微笑天使/法斗蝙蝠耳 |
| 大型犬 | 金毛、拉布拉多、德牧、阿拉斯加 | 大 | 金毛长毛温柔/拉布拉多圆润憨厚/德牧立耳英武/阿拉斯加蓝眼厚重 |

**重新随机机制**：
- 用户对生成的狗狗不满意 → 可以点击"重新生成"
- **无限次重新随机**，无限制（因为这是首次登录的一次性流程）
- 每次重新生成：犬种、颜色、装扮全部重新随机
- 确认领养后，狗狗**永久固定**，不可再更改

**装扮池设计**：

```
装扮池
├── 男孩池
│   ├── 项圈类：皮革项圈、铃铛项圈、领带、围巾
│   ├── 帽子类：棒球帽、渔夫帽、鸭舌帽、皇冠
│   └── 眼镜类：圆框眼镜、墨镜、方形眼镜
│
└── 女孩池
    ├── 项圈类：蝴蝶结、蕾丝项圈、铃铛项圈、花朵项圈
    ├── 帽子类：贝雷帽、渔夫帽、皇冠、蝴蝶结发夹
    └── 眼镜类：圆框眼镜、心形眼镜、猫耳眼镜
```

**默认装扮规则**：
- 选择"男孩" → 必定获得一件男孩池项圈类装扮
- 选择"女孩" → 必定获得一件女孩池项圈类装扮
- 后续解锁装扮时，从对应性别池中随机

#### 3.2.5 领养仪式

**流程**：
1. 狗狗生成动画（动森风格渐显，2.5D 等距视角展示）
2. 展示狗狗信息卡：
   - 品种名
   - 随机名字建议（可修改）
   - 性格标签（基于品种 + 随机）
   - 外观展示（2.5D 等距视角）
3. 用户确认命名
4. "领养成功"动画（撒花/烟花效果）
5. 进入主界面

**命名规则**：
- 系统随机推荐 3 个名字供选择
- 用户可自定义输入（2-8 个字符）
- 命名后可在设置中修改

### 3.3 24 小时场景系统

#### 3.3.1 设计理念

场景不是固定的，而是**基于真实时间动态变化**。用户在不同时间打开 App，会看到狗狗在做不同的事情，处于不同的场景中。

**核心规则**：
- 场景精确到**小时**级别
- 每个时段设计 **5-8 个场景变体**，共计 **170+ 个变体**
- 同一时段的不同变体，**布局必须明显变化**（不只是换动作/道具）
- 不同地点 / 不同空间结构 / 不同视角焦点 / 不同天气表现，至少满足一项
- 不同生活模式有完全不同的时间线和场景池

**双层镜头规则**：
1. **默认 — 全貌**：展示整个房间/场景的完整布局，狗狗在环境中
2. **点击狗狗 — 聚焦**：镜头推进到狗狗周围小区域，显示细节和可交互元素

**场景触发机制**：
- **App 启动 / 回到前台**：读取当前真实时间 → 确定时段 → 加权随机选变体
- **App 内 Timer**：每 30 分钟检查一次，时段变化时自动切换场景
- **天气影响**：仅影响自由人模式的室外/室内概率

#### 3.3.5 场景过渡机制

**自然过渡**（用户重新打开 App）：
- 根据当前真实时间 → 确定时段 → 加权随机选变体 → 渲染场景
- 不需要手动切换

**加速过渡**（用户主动加速时间，详见 3.4）：
- 播放动态蒙太奇动画（3 秒内展示多个场景变化）
- 例如：8:00 加速 → 播放 7:00-8:00 的场景（洗漱 → 地铁 → 到公司）

**随机性规则**：
- 同一时段内，从变体池中加权随机选择
- 避免连续两天同一时段出现相同变体
- 某些变体有触发条件（天气、概率判定等）

### 3.4 时间加速系统

#### 3.4.1 核心机制

用户可以主动加速时间流逝，快速跳过当前场景。

**规则**：
- **冷却时间**：3 小时（每次加速后开始计时）
- **冷却显示**：隐藏！只有用户尝试加速时才提示"还需等待 X 小时"
- **加速动画**：快速蒙太奇（3 秒内展示多个场景变化）
- **加速范围**：从当前时间加速到下一个"有意义的时间点"

#### 3.4.2 睡觉锁定

**规则**：
- 到达各模式的睡觉时间后，加速按钮变为**灰色不可点击**
- 显示文字："狗狗睡觉了 💤"
- 不能通过加速跳过睡觉时间

**睡觉时间**：
- 上班族：23:00 后锁定
- 自由人：2:00 后锁定
- 自律者：22:00 后锁定

#### 3.4.3 解锁机制

**基于真实时间解锁**（不是游戏内时间）：

| 生活模式 | 每日解锁时间 | 说明 |
|---------|------------|------|

**解锁含义**：

### 3.10 狗狗系统（核心系统）

#### 3.10.1 属性系统

详见 **3.5 属性系统**。狗狗的四维属性（心情/活力/饱腹/清洁）驱动所有行为表现和视觉反馈。

#### 3.10.2 性格系统

**性格 = f(品种天性, 用户模式, 随机特质)**

| 用户模式 | 狗狗性格倾向 | 行为特征 |
|---------|-------------|---------|
| 上班族 | 规律型 | 固定时间睡觉/吃饭，会在"下班时间"兴奋 |
| 自由人 | 随性型 | 行为不可预测，经常做奇怪的事 |
| 自律者 | 活力型 | 精力充沛，喜欢运动，早起 |

**随机特质**（每只狗随机 1-2 个）：
- 胆小/勇敢
- 粘人/独立
- 贪吃/挑食
- 懒洋洋/精力旺盛
- 聪明/憨憨

#### 3.10.3 行为系统

**行为分类**：

1. **自主行为**（无需用户触发，基于状态和时间自动发生）
   - 睡觉、发呆、舔毛、打哈欠
   - 找食物、喝水
   - 玩耍（自己玩玩具）
   - 看窗外、晒太阳

2. **反应行为**（用户互动触发）
   - 被抚摸：摇尾巴、眯眼、发出满足声
   - 被喂食：开心吃饭、挑食表现
   - 被叫名字：抬头、跑过来
   - 长时间未互动：委屈眼神、小声呜咽

3. **特殊行为**（基于条件触发）
   - 用户打开 App：根据时间段有不同反应
     - 早上：刚睡醒，伸懒腰
     - 中午：可能刚吃完饭在休息
     - 晚上：兴奋迎接
     - 深夜：已经睡了，被吵醒有点迷糊
   - 天气变化：下雨天可能更想窝着
   - 亲密度高：更多撒娇行为

#### 3.10.4 成长系统

**年龄系统**：
- 狗狗有年龄概念（幼犬 → 成年犬 → 老年犬）
- 年龄增长：1 现实月 = 1 狗年
- 不同年龄段：外观微调、行为变化

**亲密度等级**：

| 等级 | 亲密度 | 解锁内容 |
|------|--------|---------|
| Lv.1 陌生 | 0-20 | 基础互动 |
| Lv.2 熟悉 | 21-40 | 更多表情反应 |
| Lv.3 喜欢 | 41-60 | 特殊互动动作 |
| Lv.4 亲密 | 61-80 | 装扮系统完全解锁 |
| Lv.5 挚爱 | 81-100 | 稀有装扮、特殊场景 |

#### 3.10.5 外观系统

**外观组成**：

```
狗狗外观渲染
├── 基础层：品种体型 + 毛色
├── 细节层：眼睛、鼻子、特殊标记
├── 装扮层：项圈/帽子/眼镜（可更换）
└── 状态层：表情、姿态（基于当前状态）
```

**装扮获取方式**：
- 初始：随机 1 件
- 亲密度等级解锁
- 特殊日期赠送（生日、节日）
- 未来可扩展：任务奖励、商店购买

## 四、技术架构

### 4.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 框架 | SwiftUI | 原生 iOS 开发 |
| 最低版本 | **iOS 17.0+** | 使用 @Observable 宏 |
| 架构模式 | **MVVM + Services** | View → ViewModel(@Observable) → Service |
| 状态管理 | **@Observable** | iOS 17 新宏，替代 ObservableObject |
| 场景渲染 | **SwiftUI Canvas + TimelineView** | 2.5D 等距场景渲染，不依赖 SpriteKit |
| 数据持久化 | **Codable JSON + 原子写入** | 树形数据，无需关系数据库 |
| 动画 | SwiftUI Animation + Canvas | 场景内动画 + UI 过渡动画 |
| 网络 | 无 | 纯本地，无网络依赖 |

### 4.2 技术决策说明

| 决策 | 选择 | 原因 |
|------|------|------|
| 状态管理 | @Observable 而非 ObservableObject | iOS 17+ 更简洁，自动追踪属性访问，性能更好 |
| 场景渲染 | Canvas + TimelineView 而非 SpriteKit | 2.5D 等距视图不需要物理引擎，Canvas 足够且更轻量 |
| 持久化 | JSON + 原子写入而非 SwiftData | 数据是树形结构（一只狗 + 配置），不需要关系查询 |
| 架构 | MVVM + Services 而非 TCA | 项目规模适中，MVVM 足够清晰，无过度设计 |

## 五、设计规范

### 5.1 视觉风格

**2.5D 等距视角 + 像素风**

| 元素 | 规范 |
|------|------|
| 渲染方式 | 2.5D 等距视角（isometric），Canvas 绘制 |
| 风格参考 | 动物森友会（温暖、可爱、治愈） |
| 像素规格 | 角色 32×32 基准，场景道具按比例 |
| 调色板 | 温暖大地色系，低饱和度 |
| 动画 | 帧动画（精灵图序列）+ Canvas 插值移动 |
| UI 组件 | 像素风直角 + 硬阴影，与场景风格统一 |

**设计理念**：
- 2.5D 等距视角让场景有立体感，但不复杂
- 像素风带来怀旧温暖的质感
- 色彩柔和，让人感到放松和舒适

**主色调**：

```
背景色：#FFF8E8（暖米色）
主色调：#5D8B6A（森林绿）
强调色：#C69A3E（暖金色）
文字色：#26382B（深绿黑）
```

### 5.2 交互原则

1. **简单**：最多 2 步完成任何操作
2. **即时**：点击后立即有反馈（动画/音效）
3. **温柔**：没有警告、没有惩罚、没有负面反馈
4. **有趣**：每个互动都有小惊喜

### 5.3 音效设计（MVP 后）

| 场景 | 音效 |
|------|------|
| 抚摸 | 满足的"嘤"声 |
| 喂食 | 咀嚼声 |
| 场景切换 | 轻柔过渡音 |
| 睡觉 | 轻微的呼吸声 |
| 开心 | 欢快的叫声 |

### 6.4 场景状态

```swift
struct SceneState: Codable {
    var currentSceneId: String        // 当前场景变体 ID（如 "W-9-3"）
    var currentTimeSlot: TimeSlot     // 当前时段
    var currentWeather: Weather       // 当前天气
    var lastSceneChangeTime: Date     // 上次场景切换时间
    var lastVariantId: String         // 上一个变体 ID（避免重复）
}

enum TimeSlot: String, Codable, CaseIterable {
    case earlyMorning = "清晨"       // 5:00-7:00
    case morning = "上午"            // 7:00-11:00
    case noon = "中午"               // 11:00-14:00
    case afternoon = "下午"          // 14:00-18:00
    case evening = "傍晚"            // 18:00-21:00
    case night = "夜晚"              // 21:00-24:00
    // 自由人专用
    case lateMorning = "上午晚"      // 10:00-12:00
    case lateAfternoon = "下午晚"    // 14:00-18:00
    case lateNight = "深夜"          // 0:00-2:00
}

enum Weather: String, Codable {
    case sunny, cloudy, rainy, thunderstorm, snowy
}

struct SceneVariant: Codable {
    let id: String                    // 如 "W-9-3", "F-14-R1"
    let timeSlot: TimeSlot
    let layout: SceneLayout           // 布局描述

    let interactiveProps: [InteractiveProp] // 可交互道具
    let attributeEffects: [String: Int]     // 每小时属性变化
    let weatherCondition: Weather?    // 天气条件（nil=任意天气）
}
```

## 九、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| 像素动画工作量大 | 开发周期延长 | 优先实现核心动画，其余迭代 |
| 行为引擎复杂 | 技术难度高 | MVP 先实现简单规则，后续优化 |
| 用户新鲜感过后流失 | 留存下降 | 持续更新内容（装扮/场景/活动） |
| 与市面宠物 App 同质化 | 差异化不足 | 强调"生活镜像"概念，不是养成游戏 |
