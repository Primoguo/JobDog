# Job Dog v2.0 - 产品需求文档（PRD）

**版本**：2.0  
**产品类型**：原生 iOS 情感陪伴 App  
**目标平台**：iOS 15.0+（适配 iPhone SE 至 iPhone 15 Pro Max）  
**文档日期**：2026-06-26  
**状态**：待审核

---

## 一、产品概述

### 1.1 产品定位

Job Dog 是一款面向当代年轻人和中年人的**情感陪伴类 App**。在信息过载、快节奏生活的当下，用户打开 Job Dog 不是为了打卡、不是为了完成任务，而是为了**逃离千篇一律的快娱乐环境**，获得真正的放松和简单的陪伴。

### 1.2 核心理念

| 维度 | 传统习惯类 App | Job Dog v2.0 |
|------|---------------|-------------|
| 驱动力 | 任务驱动、目标导向 | 情感驱动、陪伴导向 |
| 用户感受 | "我应该做..." | "我想看看我的狗..." |
| 交互频率 | 定时打卡 | 随时打开，随时陪伴 |
| 情感连接 | 弱（工具属性） | 强（情感投射） |
| 核心体验 | 成就感 | 放松感、被需要感 |

### 1.3 一句话描述

> **Job Dog 是你的数字宠物，也是你的生活镜像。它不需要你做什么，只是陪着你。**

## 二、目标用户

### 2.1 用户画像

**主要用户群体**：25-45 岁的城市青年和中年人

| 用户类型 | 特征 | 核心需求 |
|---------|------|---------|
| 上班族 | 朝九晚六，生活规律但千篇一律 | 下班后的放松，回家有人"等"的感觉 |
| 自由职业/无业 | 生活无拘无束，但可能感到迷茫 | 稳定的陪伴感，日常的小确幸 |
| 自律型 | 严格作息，爱健身/徒步等 | 共同成长的伙伴，不评判的陪伴 |

### 2.2 用户场景

**场景 1：上班族下班回家**
- 疲惫地打开 App
- 狗狗根据时间做出反应（可能是刚睡醒、可能在发呆、可能在门口等你）
- 简单互动，摸摸头，喂点吃的
- 感受：有人等我回来了

**场景 2：自由职业者午后**
- 工作间隙打开 App
- 狗狗可能在晒太阳、可能在玩耍
- 一起"出去走走"（散步功能）
- 感受：有个伴儿真好

**场景 3：自律型用户晨跑后**
- 运动完打开 App
- 狗狗也"刚运动完"（状态同步）
- 一起喝水、休息
- 感受：我们一起在生活

### 3.2 首次体验流程（Onboarding）

#### 完整流程

```
欢迎页 → 生活模式选择 → 性别选择 → 随机生成狗狗 → 领养仪式（命名）→ 主界面
```

**关键原则**：
- 犬种**不可选**，完全随机生成
- 性别选择**只影响装扮风格**，不影响犬种/颜色
- 一旦确认领养，狗狗**不可更改**（命运感）
- 支持存档，下次打开自动加载

---

#### 3.2.1 欢迎页

**界面元素**：
- APP Logo（动森风格插画）
- 粗体 "Job Dog" 标题
- "开始" 按钮
- 如有存档，显示"继续游戏"选项

**交互**：
- 点击"开始" → 进入生活模式选择
- 点击"继续游戏" → 直接进入主界面（跳过 Onboarding）

---

#### 3.2.2 生活模式选择

**设计原则**：不是"你是什么人"，而是"你希望你的生活节奏是怎样的"

**交互设计**：
- 三个卡片**纵向堆叠**排列
- 每个卡片包含：图标 + 名称 + 描述 + 小场景预览（动森风格）
- 点击卡片选中，可返回修改

| 模式 | 图标 | 描述文案 | 狗狗特征倾向 |
|------|------|---------|-------------|
| 🏢 上班族 | 公文包 | 朝九晚六，规律但安稳 | 狗狗更规律，会在固定时间等你 |
| 🌊 自由人 | 冲浪板 | 随性而为，不被定义 | 狗狗更随性，随时可能做奇怪的事 |
| 🏃 自律者 | 跑鞋 | 严格作息，追求更好的自己 | 狗狗更活力，喜欢运动和挑战 |

---

#### 3.2.3 性别选择

**说明**：性别**不是狗狗的性别**，而是决定狗狗的**装扮风格方向**。

| 选择 | 装扮风格 | 默认装扮 |
|------|---------|---------|
| 男孩 | 偏男性化 | 领带/棒球帽 |
| 女孩 | 偏女性化 | 蝴蝶结/贝雷帽 |

**影响范围**：
- ✅ 影响：默认装扮、可解锁的装扮池
- ❌ 不影响：犬种、毛色、性格、行为

---

#### 3.2.4 随机生成狗狗（核心创新点）

**完全随机生成**：
- 犬种：从 20 个犬种中随机
- 毛色：从品种标准色库中随机
- 眼睛颜色：随机（小概率异色瞳）
- 特殊标记：小概率出现斑点/花纹
- 初始装扮：从**性别对应的装扮池**中随机

**20 个犬种池**：

| 类别 | 犬种 | 体型 |
|------|------|------|
| 小型犬 | 柴犬、柯基、比熊、博美、吉娃娃、约克夏 | 小 |
| 中型犬 | 边牧、哈士奇、萨摩耶、法斗、雪纳瑞、中华田园犬 | 中 |
| 大型犬 | 金毛、拉布拉多、德牧、阿拉斯加、萨路基猎犬、阿富汗猎犬 | 大 |

**重新随机机制**：
- 用户对生成的狗狗不满意 → 可以点击"重新生成"
- **无限次重新随机**，无限制（因为这是首次登录的一次性流程）
- 每次重新生成：犬种、颜色、装扮全部重新随机
- 确认领养后，狗狗**永久固定**，不可再更改

**装扮池设计**：

```
装扮池
├── 男孩池
│   ├── 项圈类：皮革项圈、铃铛项圈、领带、围巾
│   ├── 帽子类：棒球帽、渔夫帽、鸭舌帽、皇冠
│   └── 眼镜类：圆框眼镜、墨镜、方形眼镜
│
└── 女孩池
    ├── 项圈类：蝴蝶结、蕾丝项圈、铃铛项圈、花朵项圈
    ├── 帽子类：贝雷帽、渔夫帽、皇冠、蝴蝶结发夹
    └── 眼镜类：圆框眼镜、心形眼镜、猫耳眼镜
```

**默认装扮规则**：
- 选择"男孩" → 必定获得一件男孩池项圈类装扮
- 选择"女孩" → 必定获得一件女孩池项圈类装扮
- 后续解锁装扮时，从对应性别池中随机

---

#### 3.2.5 领养仪式

**流程**：
1. 狗狗生成动画（动森风格渐显，2.5D 等距视角展示）
2. 展示狗狗信息卡：
   - 品种名
   - 随机名字建议（可修改）
   - 性格标签（基于品种 + 随机）
   - 外观展示（2.5D 等距视角）
3. 用户确认命名
4. "领养成功"动画（撒花/烟花效果）
5. 进入主界面

**命名规则**：
- 系统随机推荐 3 个名字供选择
- 用户可自定义输入（2-8 个字符）
- 命名后可在设置中修改

### 3.3 24 小时场景系统

#### 3.3.1 设计理念

场景不是固定的，而是**基于真实时间动态变化**。用户在不同时间打开 App，会看到狗狗在做不同的事情，处于不同的场景中。

**核心规则**：
- 场景精确到**小时**级别
- 每个时间段设计 **5-8 个不重样的场景变体**
- 在一定规则内保持**高自由度**（如：上班族可能早到也可能迟到）
- 不同生活模式有完全不同的时间线和场景池

#### 3.3.2 上班族时间线（6:00 - 23:00）

| 时间 | 场景 | 狗狗行为 | 变体示例 |
|------|------|---------|---------|
| 6:00-7:00 | 家中浴室 | 洗漱 | 刷牙、洗脸、照镜子、吹头发 |
| 7:00-8:00 | 地铁/公交 | 通勤 | 看手机、打瞌睡、让座、被挤 |
| 8:00-9:00 | 办公室工位 | 到公司 | 开电脑、泡咖啡、整理桌面、迟到赶路 |
| 9:00-12:00 | 办公室工位 | 工作 | 打字、开会、摸鱼、喝水、伸懒腰 |
| 12:00-13:00 | 公司食堂/餐厅 | 午饭 | 吃饭、排队、和同事聊天、点外卖 |
| 13:00-14:00 | 办公室/休息区 | 午休 | 趴桌子、闭眼休息、散步消食 |
| 14:00-18:00 | 办公室工位 | 下午工作 | 打字、开会、摸鱼、看窗外、吃零食 |
| 18:00-19:00 | 地铁/公交 | 下班通勤 | 回家、逛超市、走路、骑车 |
| 19:00-20:00 | 家中厨房/客厅 | 晚饭 | 做饭、吃饭、点外卖、看电视 |
| 20:00-22:00 | 家中客厅/卧室 | 放松 | 沙发躺着、刷手机、洗澡、看书 |
| 22:00-23:00 | 卧室 | 准备睡觉 | 刷牙、换睡衣、关灯、玩手机 |

**特殊机制**：
- **10% 概率失眠**：22:00-23:00 有 10% 概率触发"失眠"场景，狗狗在床上翻来覆去
- **迟到/早到随机性**：8:00-9:00 时间段，随机决定是"准时到工位"还是"迟到赶路"

#### 3.3.3 自由人时间线（10:00 - 2:00）

**特点**：时间不固定，活动在大致范围内随机

| 时间范围 | 可能的场景 | 随机性说明 |
|---------|-----------|-----------|
| 10:00-12:00 | 卧室/家中 | 随机醒来时间，可能赖床、可能早就起了 |
| 11:00-15:00 | 任意地点 | 午饭时间完全随机，可能在家做、可能出去吃 |
| 全天可触发 | 城市溜达、商场、咖啡馆、公园、家里躺着 | 无固定顺序，随机选择 |
| 18:00-22:00 | 餐厅/家中 | 晚饭时间随机 |
| 22:00-2:00 | 各种夜间场景 | 熬夜、看剧、打游戏、出门散步 |

**不可能出现的场景**：
- ❌ 上班、开会、打卡（自由人不会做这些事）
- ✅ 睡懒觉、突然出门、临时决定做某事

#### 3.3.4 自律者时间线（5:00 - 22:00）

**特点**：完全固定，每天在固定时间做固定事

| 时间 | 场景 | 狗狗行为 |
|------|------|---------|
| 5:00-5:30 | 卧室 | 起床 |
| 5:30-6:30 | 户外/公园 | 晨跑 |
| 6:30-7:00 | 家中浴室 | 洗漱 |
| 7:00-7:30 | 家中厨房 | 早餐 |
| 7:30-12:00 | 书房/工作室 | 工作/学习 |
| 12:00-12:30 | 家中厨房 | 午饭 |
| 12:30-13:00 | 客厅 | 休息 |
| 13:00-17:00 | 书房/工作室 | 工作/学习 |
| 17:00-18:00 | 户外/健身房 | 运动 |
| 18:00-18:30 | 家中浴室 | 洗澡 |
| 18:30-19:00 | 家中厨房 | 晚饭 |
| 19:00-21:00 | 客厅/书房 | 阅读/学习 |
| 21:00-22:00 | 卧室 | 准备睡觉 |

**不可能出现的场景**：
- ❌ 熬夜、赖床、临时改变计划
- ✅ 严格遵循时间表，风雨无阻

#### 3.3.5 场景过渡机制

**自然过渡**（用户重新打开 App）：
- 根据当前真实时间，自动显示对应场景
- 不需要手动切换

**加速过渡**（用户主动加速时间，详见 3.3.6）：
- 播放动态蒙太奇动画
- 快速展示从"当前时间"到"目标时间"的场景变化
- 例如：8:00 加速 → 播放 7:00-8:00 的场景（洗漱 → 地铁 → 到公司，3 秒内完成）

**随机性规则**：
- 同一时间段内，从 5-8 个变体中随机选择
- 避免连续两天同一时间段出现相同变体
- 某些变体有触发条件（如：迟到只在 8:00-9:00 出现）

---

### 3.4 时间加速系统

#### 3.4.1 核心机制

用户可以主动加速时间流逝，快速跳过当前场景。

**规则**：
- **冷却时间**：3 小时（每次加速后开始计时）
- **冷却显示**：隐藏！只有用户尝试加速时才提示"还需等待 X 小时"
- **加速动画**：快速蒙太奇（3 秒内展示多个场景变化）
- **加速范围**：从当前时间加速到下一个"有意义的时间点"

#### 3.4.2 睡觉锁定

**规则**：
- 到达各模式的睡觉时间后，加速按钮变为**灰色不可点击**
- 显示文字："狗狗睡觉了 💤"
- 不能通过加速跳过睡觉时间

**睡觉时间**：
- 上班族：23:00 后锁定
- 自由人：2:00 后锁定
- 自律者：22:00 后锁定

#### 3.4.3 解锁机制

**基于真实时间解锁**（不是游戏内时间）：

| 生活模式 | 每日解锁时间 | 说明 |
|---------|------------|------|
| 自律者 | 5:00 | 最早解锁，符合早起人设 |
| 上班族 | 6:00 | 中等时间 |
| 自由人 | 10:00 | 最晚解锁，符合赖床人设 |

**解锁含义**：
- 每天到了对应时间，加速功能才可用
- 例如：上班族在 6:00 前，即使冷却结束，也不能加速

### 3.5 狗狗系统（核心系统）

#### 3.3.1 状态系统

**四维状态模型**：

| 状态 | 范围 | 影响因素 | 表现 |
|------|------|---------|------|
| 心情 | 0-100 | 互动、时间、环境 | 表情、动作、叫声 |
| 饥饿 | 0-100 | 时间、喂食 | 找食物、可怜眼神 |
| 精力 | 0-100 | 时间、睡眠、玩耍 | 活跃度、睡觉频率 |
| 亲密度 | 0-100（只增不减） | 互动频率、照顾质量 | 服从度、撒娇频率 |

**状态变化规则**：

```
心情：
- 基础衰减：-1/小时
- 互动加成：+5~+15（取决于互动类型）
- 长时间未互动：-2/小时（>8小时）

饥饿：
- 基础增长：+2/小时
- 喂食后：-30~-50（取决于食物）
- 饥饿>80：心情额外 -1/小时

精力：
- 自然恢复：+3/小时（休息状态）
- 活动消耗：-5~15/次互动
- 夜间（22:00-06:00）：恢复速度 ×2

亲密度：
- 每日互动：+2~+5
- 连续互动天数：额外加成
- 忽视（>24小时未打开）：不减少，但增长暂停
```

#### 3.3.2 性格系统

**性格 = f(品种天性, 用户模式, 随机特质)**

| 用户模式 | 狗狗性格倾向 | 行为特征 |
|---------|-------------|---------|
| 上班族 | 规律型 | 固定时间睡觉/吃饭，会在"下班时间"兴奋 |
| 自由人 | 随性型 | 行为不可预测，经常做奇怪的事 |
| 自律者 | 活力型 | 精力充沛，喜欢运动，早起 |

**随机特质**（每只狗随机 1-2 个）：
- 胆小/勇敢
- 粘人/独立
- 贪吃/挑食
- 懒洋洋/精力旺盛
- 聪明/憨憨

#### 3.3.3 行为系统

**行为分类**：

1. **自主行为**（无需用户触发，基于状态和时间自动发生）
   - 睡觉、发呆、舔毛、打哈欠
   - 找食物、喝水
   - 玩耍（自己玩玩具）
   - 看窗外、晒太阳

2. **反应行为**（用户互动触发）
   - 被抚摸：摇尾巴、眯眼、发出满足声
   - 被喂食：开心吃饭、挑食表现
   - 被叫名字：抬头、跑过来
   - 长时间未互动：委屈眼神、小声呜咽

3. **特殊行为**（基于条件触发）
   - 用户打开 App：根据时间段有不同反应
     - 早上：刚睡醒，伸懒腰
     - 中午：可能刚吃完饭在休息
     - 晚上：兴奋迎接
     - 深夜：已经睡了，被吵醒有点迷糊
   - 天气变化：下雨天可能更想窝着
   - 亲密度高：更多撒娇行为

#### 3.3.4 成长系统

**年龄系统**：
- 狗狗有年龄概念（幼犬 → 成年犬 → 老年犬）

- 不同年龄段：外观微调、行为变化

**亲密度等级**：

| 等级 | 亲密度 | 解锁内容 |

### 4.3 核心算法

#### 4.3.1 狗狗生成算法

```swift
struct DogGenerator {
    static func generate(breed: Breed, lifeMode: LifeMode) -> Dog {
        let seed = Int.random(in: 0...Int.max)
        var rng = SeededRandomNumberGenerator(seed: seed)
        
        // 随机外观
        let mainColor = breed.standardColors.randomElement(using: &rng)!
        let eyeColor = EyeColor.random(for: breed, using: &rng)
        let accessory = Accessory.random(using: &rng)
        
        // 性格（基于品种 + 生活模式 + 随机）
        let personality = Personality.generate(
            breed: breed,
            lifeMode: lifeMode,
            rng: &rng
        )
        
        return Dog(
            id: UUID(),
            breed: breed,
            name: "",
            mainColor: mainColor,
            eyeColor: eyeColor,
            accessory: accessory,
            personality: personality,
            lifeMode: lifeMode,
            createdAt: Date()
        )
    }
}
```

#### 4.3.2 行为决策引擎

```swift
class BehaviorEngine {
    func decideNextAction(for dog: Dog) -> DogAction {
        let context = BehaviorContext(
            time: Date(),
            dogStatus: dog.status,
            personality: dog.personality,
            lastAction: dog.lastAction,
            environment: currentEnvironment
        )
        
        // 优先级：生理需求 > 情绪需求 > 自主行为
        if dog.status.hunger > 80 {
            return .seekFood
        }
        if dog.status.energy < 20 {
            return .sleep
        }
        
        // 基于性格和时间的行为选择
        let possibleActions = BehaviorLibrary.getPossibleActions(context: context)
        return possibleActions.weightedRandom()
    }
}
```

## 五、设计规范

### 5.1 视觉风格

**动物森友会风格 + 2.5D 等距视角**

| 元素 | 规范 |
|------|------|
| 渲染方式 | 2.5D 等距视角（isometric） |
| 风格参考 | 动物森友会（温暖、可爱、治愈） |
| 调色板 | 温暖大地色系为主 |
| 动画 | 流畅自然，非像素化 |
| 圆角 | 柔和圆角（UI 组件） |
| 阴影 | 柔和阴影，营造立体感 |

**设计理念**：
- 不是像素风的冷硬感，而是动森风的温暖治愈
- 2.5D 等距视角让场景有立体感，但不复杂
- 色彩柔和，让人感到放松和舒适

**主色调**：

```
背景色：#FFF8E8（暖米色）
主色调：#5D8B6A（森林绿）
强调色：#C69A3E（暖金色）
文字色：#26382B（深绿黑）
```

### 5.2 交互原则

1. **简单**：最多 2 步完成任何操作
2. **即时**：点击后立即有反馈（动画/声音）
3. **温柔**：没有警告、没有惩罚、没有负面反馈
